01 · introducción

¿Qué es HTML5?

HTML son las siglas de HyperText Markup Language — el lenguaje que usan los navegadores (Chrome, Firefox, Edge) para mostrar páginas web. HTML5 es su versión más moderna, lanzada en 2014.

🏠 Analogía: una página web es como una casa
HTML → La estructura (paredes, puertas, ventanas)
CSS → La decoración (colores, muebles, estilo)
JS → La electricidad (lo que hace las cosas funcionar)

El flujo es simple: escribes código en un archivo .html → lo abres en un navegador → el navegador lo convierte en una página visual.

02 · fundamentos

Estructura base

Todo documento HTML5 comienza con esta misma plantilla. Memorízala, la usarás siempre.

html
<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi primera página</title>
  </head>
  <body>
    <!-- Aquí va todo lo que el usuario puede VER -->
    <h1>¡Hola, mundo!</h1>
  </body>
</html>
🔑

Regla de oro: La mayoría de etiquetas tienen apertura <p> y cierre </p>. El cierre siempre lleva una barra /.

03 · referencia

Etiquetas esenciales

Estas son las etiquetas que usarás en el 80% de tus proyectos. Explóralas una por una.

<html>

La raíz de todo el documento. Contiene absolutamente todo. El atributo lang indica el idioma.

contenedor raíz
<head>

Sección invisible con metadatos: título, estilos, configuración del documento.

invisible
<body>

Todo lo que el usuario ve en pantalla va dentro de <body>. Solo hay uno por página.

visible
<h1> – <h6>

Títulos del más grande (h1) al más pequeño (h6). Usa solo un <h1> por página.

estructura
<p>

Párrafo de texto. La etiqueta más usada para contenido escrito. Crea espacio automático entre párrafos.

texto
<a>

Enlace. Requiere el atributo href con la URL. Con target="_blank" abre en pestaña nueva.

navegación
<img>

Imagen. Es de cierre automático. Requiere src (URL) y alt (descripción).

auto-cierre
<hr> <br>

hr es una línea separadora. br es un salto de línea. Ambas son de cierre automático.

auto-cierre

La etiqueta <a> en detalle

html
<!-- Enlace básico -->
<a href="https://www.google.com">Ir a Google</a>

<!-- Abre en pestaña nueva -->
<a href="https://github.com" target="_blank">GitHub</a>

<!-- Enlace de correo -->
<a href="mailto:hola@ejemplo.com">Envíame un correo</a>

La etiqueta <img> en detalle

html
<!-- Imagen desde internet -->
<img
  src="https://picsum.photos/400/300"
  alt="Descripción de la imagen"
  width="400"
>

<!-- Imagen local -->
<img src="foto.jpg" alt="Mi foto">
♿

¿Por qué es importante el alt? Las personas que usan lectores de pantalla dependen de este texto para saber qué hay en la imagen. Es obligatorio por accesibilidad y SEO.

04 · práctica

🛠️ Taller: Mi primera página web

Objetivo

Crear una página web personal con tu nombre, una foto o avatar, una descripción y al menos un enlace. Tienes 45 minutos.

1

Crea tu archivo

Abre VS Code o CodePen y crea un nuevo archivo llamado index.html. Escribe la estructura base o cópiala de la sección anterior.

2

Completa la plantilla

Copia este código y reemplaza todo lo que está en MAYÚSCULAS con tu información real:

index.html · plantilla del taller
<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Página de TU_NOMBRE</title>
  </head>
  <body>

    <!-- SECCIÓN 1: Tu presentación -->
    <h1>Hola, soy TU_NOMBRE 👋</h1>
    <p>Tengo EDAD años y vivo en TU_CIUDAD.</p>

    <!-- SECCIÓN 2: Sobre ti -->
    <h2>Sobre mí</h2>
    <p>ESCRIBE 2 O 3 ORACIONES SOBRE TI.</p>

    <!-- SECCIÓN 3: Tu avatar -->
    <h2>Mi avatar</h2>
    <img
      src="https://api.dicebear.com/7.x/avataaars/svg?seed=TU_NOMBRE"
      alt="Avatar de TU_NOMBRE"
      width="200"
    >

    <!-- SECCIÓN 4: Intereses -->
    <h2>Mis intereses</h2>
    <p>⭐ INTERÉS 1</p>
    <p>⭐ INTERÉS 2</p>
    <p>⭐ INTERÉS 3</p>

    <!-- SECCIÓN 5: Un enlace -->
    <h2>Encuéntrame en</h2>
    <p>
      <a href="URL_RED_SOCIAL" target="_blank">
        Mi perfil en NOMBRE_RED_SOCIAL
      </a>
    </p>

    <hr>
    <p>Hecho con ❤️ por TU_NOMBRE — 2025</p>

  </body>
</html>
3

✅ Checklist antes de terminar

Verifica que tu página tenga todo esto:

  • Estructura base completa (DOCTYPE, html, head, body)
  • Título en la pestaña del navegador (<title>)
  • Al menos un <h1> con tu nombre
  • Al menos dos <h2> como subtítulos
  • Al menos tres <p> con texto
  • Una <img> con src y alt
  • Al menos un <a> con href
  • Un <hr> separador
  • Comentarios explicando las secciones

06 · colaboración

🔍 Revisión por pares

El Code Review es una práctica real usada por programadores profesionales. Sirve para encontrar errores y aprender del código de otros.

📋

Cómo funciona: Busca un compañero → intercambien sus archivos index.html → revisen usando la guía de abajo → den retroalimentación → corrijan su propio código.

¿Qué revisar?

🔴 Errores críticos
¿Todas las etiquetas tienen cierre?
¿Las etiquetas están bien escritas? (<body> no <bodi>)
¿Los atributos tienen comillas dobles?
🟡 Buenas prácticas
¿Hay un solo <h1> por página?
¿Las imágenes tienen alt descriptivo?
¿El código tiene buena indentación?
🟢 Creatividad
¿La página tiene contenido interesante?
¿Se ve organizada y fácil de leer?
¿Usó comentarios para explicar el código?
¿Usó alguna etiqueta extra?

Cómo dar retroalimentación constructiva

❌ No digas✅ Di mejor
"Tu código está mal" "En la línea 15, falta cerrar la etiqueta <p>"
"No entendiste nada" "Aquí podrías agregar alt a la imagen para mejorar la accesibilidad"
"Está perfecto, no cambies nada" "Me gustó que usaste emojis 😄 Podrías probar con <strong> para resaltar palabras"

Formulario de revisión

Usa esta plantilla para dar tu retroalimentación por escrito:

REVISIÓN DE CÓDIGO — [Revisor] → [Autor] Fecha: ___________ ✅ COSAS QUE HIZO BIEN: 1. 2. 3. 🔧 ERRORES ENCONTRADOS: 1. (Línea aprox.): 2. (Línea aprox.): 💡 SUGERENCIAS DE MEJORA: 1. 2. ⭐ PUNTUACIÓN: ___/10 💬 COMENTARIO FINAL:
