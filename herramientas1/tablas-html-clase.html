Introducción a las Tablas HTML

Entendemos qué son, para qué sirven y aprendemos las etiquetas <table>, <tr>, <td> y <th>

¿Qué es una tabla y para qué sirve?

Imagina que tienes que organizar los resultados de un torneo de videojuegos: jugadores, puntajes y posiciones. ¿Cómo lo harías en papel? Probablemente dibujarías filas y columnas. En HTML, las tablas hacen exactamente eso: organizan información en filas y columnas de forma visual y ordenada.

Horarios y calendarios

Organiza días, horas y actividades en una cuadrícula clara.

Resultados deportivos

Clasifica equipos, puntajes, partidos ganados y perdidos.

Boletas de notas

Presenta materias, períodos y promedios de forma estructurada.

Comparativas

Compara características de productos, planes o servicios.

⚠
Importante: Las tablas son para datos tabulares, no para hacer el diseño general de una página web. ¡Ese es el trabajo del CSS!

Las etiquetas fundamentales

Código de ejemplo

tabla-basica.html 
<table border="1">
  <tr>
    <th>Jugador</th>
    <th>Puntaje</th>
    <th>Nivel</th>
  </tr>
  <tr>
    <td>Carlos</td>
    <td>9800</td>
    <td>Experto</td>
  </tr>
  <tr>
    <td>Sofía</td>
    <td>11200</td>
    <td>Maestra</td>
  </tr>
</table>
💡
El atributo border="1" le agrega bordes visibles a la tabla. Sin él, las celdas no tienen borde y es difícil verlas al inicio.

Estructura semántica

Usamos <thead>, <tbody> y <tfoot> para organizar la tabla en secciones significativas

Así como un libro tiene portada, contenido e índice, una tabla HTML bien construida tiene tres secciones claras. Esto no es solo por estética: ayuda a los navegadores, lectores de pantalla y herramientas de accesibilidad a entender mejor la información.

tabla-semántica.html 
<table border="1">

  <thead>
    <tr>
      <th>Materia</th>
      <th>Nota 1</th>
      <th>Nota 2</th>
      <th>Promedio</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>Matemáticas</td>
      <td>4.2</td>
      <td>3.8</td>
      <td>4.0</td>
    </tr>
    <tr>
      <td>Lengua Castellana</td>
      <td>4.5</td>
      <td>4.7</td>
      <td>4.6</td>
    </tr>
  </tbody>

  <tfoot>
    <tr>
      <td><strong>Promedio General</strong></td>
      <td></td><td></td>
      <td><strong>4.3</strong></td>
    </tr>
  </tfoot>

</table>

Atributos: colspan y rowspan

Aprende a unir columnas con colspan y filas con rowspan para crear tablas más expresivas

colspan — Unir columnas horizontalmente

colspan le dice a una celda que ocupe el espacio de varias columnas. Se usa típicamente en títulos que deben abarcar toda la tabla.

rowspan — Unir filas verticalmente

rowspan hace que una celda ocupe el espacio de varias filas hacia abajo. Ideal para agrupar elementos por categoría.

Validación de Etiquetas Anidadas

Identificamos y corregimos los errores más frecuentes en el anidamiento de etiquetas de tabla

¿Qué es el anidamiento?

En HTML, anidar significa poner una etiqueta dentro de otra. Las tablas tienen reglas muy específicas sobre qué puede ir dentro de qué. Si las rompes, el navegador puede mostrar resultados inesperados.

Errores comunes de anidamiento

❌ Error 1 — <td> directamente dentro de <table>

❌ Incorrecto 
<table>
  <td>Esto está mal</td>
</table>
✓ Correcto 
<table>
  <tr>
    <td>Esto está bien</td>
  </tr>
</table>
💡
Las celdas <td> y <th> SIEMPRE deben estar dentro de una fila <tr>. Sin <tr>, la tabla no sabe dónde poner la celda.

❌ Error 2 — <tr> dentro de <tr>

❌ Incorrecto 
<tr>
  <td>Celda 1</td>
  <tr>  <!-- ERROR -->
    <td>Celda 2</td>
  </tr>
</tr>
✓ Correcto 
<tr>
  <td>Celda 1</td>
</tr>
<tr>
  <td>Celda 2</td>
</tr>

❌ Error 3 — <thead> fuera de lugar

❌ Incorrecto 
<table>
  <tbody>...</tbody>
  <thead>  <!-- thead al final -->
    <tr><th>...</th></tr>
  </thead>
</table>
✓ Correcto 
<table>
  <thead>  <!-- thead primero -->
    <tr><th>...</th></tr>
  </thead>
  <tbody>...</tbody>
</table>

Código 2 — Encuentra los 3 errores

8 min
🔍 Nivel medio
<table border="1">
  <tr>                       <!-- fila suelta sin thead/tbody -->
    <th colspan="2">Mi Horario</th>
  </tr>
  <tbody>
    <tr>
      <td>Lunes</td>
      <td>Matemáticas</td>
    <tr>                     <!-- </tr> olvidado arriba -->
      <td>Martes</td>
      <td>Inglés</td>
    </tr>
  </tbody>
  <thead>                   <!-- thead al final -->
    <tr>
      <th>Día</th><th>Materia</th>
    </tr>
  </thead>
</table>
Ver solución

Error 1: Primera fila <tr> flotando fuera de cualquier sección semántica.
Error 2: La segunda fila no está cerrada (falta </tr>).
Error 3: El <thead> está al final del código cuando debería ir al principio.

✓ Solución
<table border="1">
  <thead>
    <tr>
      <th colspan="2">Mi Horario</th>
    </tr>
    <tr><th>Día</th><th>Materia</th></tr>
  </thead>
  <tbody>
    <tr><td>Lunes</td><td>Matemáticas</td></tr>
    <tr><td>Martes</td><td>Inglés</td></tr>
  </tbody>
</table>

Lista de verificación — Antes de entregar cualquier tabla

  • La tabla comienza con <table> y termina con </table>
  • Usé <thead>, <tbody> y <tfoot> correctamente
  • Cada sección tiene al menos una fila <tr>
  • Todas las <td> y <th> están dentro de un <tr>
  • Todos los <tr> están dentro de <thead>, <tbody> o <tfoot>
  • Cerré correctamente todas las etiquetas con </etiqueta>
  • El número de mi colspan coincide con las columnas reales que cubre
  • Usé <th> para encabezados y <td> para datos
